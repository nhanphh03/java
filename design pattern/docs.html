Creational

Nhóm Khởi Tạo

Các pattern kiểm soát cách thức tạo ra object — giúp tách logic khởi tạo ra khỏi phần còn lại của chương trình, tăng tính linh hoạt và tái sử dụng.

🔒
Singleton
Đảm bảo chỉ có một instance duy nhất
+
Mô tả

Singleton đảm bảo một class chỉ có đúng một instance trong toàn bộ vòng đời của ứng dụng, đồng thời cung cấp một điểm truy cập global duy nhất đến instance đó.

Khi nào dùng
Database connection pool — mở nhiều connection tốn tài nguyên, chỉ cần 1 pool dùng chung.
Configuration manager — cài đặt ứng dụng chỉ cần đọc 1 lần, chia sẻ toàn app.
Logger — ghi log từ nhiều nơi nhưng phải đến cùng một file/stream.
Ví dụ code
JavaScript 
class DatabaseConnection {
  static #instance = null;
  #pool = [];

  // Constructor private — không cho new trực tiếp
  constructor() {
    if (DatabaseConnection.#instance) {
      throw new Error('Dùng getInstance() thay vì new');
    }
    this.#pool = this.createPool();
  }

  static getInstance() {
    if (!DatabaseConnection.#instance) {
      DatabaseConnection.#instance = new DatabaseConnection();
    }
    return DatabaseConnection.#instance;
  }

  query(sql) {
    // dùng connection từ pool
    const conn = this.#pool.pop();
    return conn.execute(sql);
  }
}

// ✅ Sử dụng
const db1 = DatabaseConnection.getInstance();
const db2 = DatabaseConnection.getInstance();
console.log(db1 === db2); // true — cùng 1 instance
🏭
Factory Method
Giao việc tạo object cho subclass quyết định
+
Mô tả

Định nghĩa một interface để tạo object, nhưng để các subclass quyết định class nào sẽ được khởi tạo. Factory Method cho phép class "trì hoãn" việc tạo object cho subclass.

Khi nào dùng
Không biết trước loại object cần tạo cho đến runtime.
Muốn subclass kiểm soát việc tạo object (ví dụ: framework UI tạo button cho từng platform).
Tạo notification system: Email, SMS, Push — cùng interface, khác implementation.
Ví dụ code
JavaScript
// Product interface
class Notification {
  send(message) { throw new Error('Phải implement send()'); }
}

class EmailNotification extends Notification {
  send(message) { console.log(`📧 Email: ${message}`); }
}

class SMSNotification extends Notification {
  send(message) { console.log(`📱 SMS: ${message}`); }
}

// Creator với factory method
class NotificationFactory {
  static create(type) {
    const map = {
      email: EmailNotification,
      sms: SMSNotification,
    };
    const Class = map[type];
    if (!Class) throw new Error(`Unknown type: ${type}`);
    return new Class();
  }
}

// ✅ Sử dụng
const notifier = NotificationFactory.create('email');
notifier.send('Chào mừng bạn!'); // 📧 Email: Chào mừng bạn!
🧱
Builder
Xây dựng object phức tạp từng bước
+
Mô tả

Builder tách việc xây dựng một object phức tạp ra khỏi phần biểu diễn của nó. Cho phép tạo các biểu diễn khác nhau của cùng một quá trình xây dựng, sử dụng pattern method chaining rất phổ biến.

Khi nào dùng
Object có nhiều tham số tùy chọn — tránh "telescoping constructor" anti-pattern.
Cần tạo nhiều biến thể của object (QueryBuilder, EmailBuilder, HTMLBuilder).
Muốn code dễ đọc kiểu fluent API: new Query().select('*').from('users').where(...).
Ví dụ code
JavaScript
class QueryBuilder {
  #query = { table: '', fields: ['*'], conditions: [], limit: null };

  from(table) { this.#query.table = table; return this; }

  select(...fields) { this.#query.fields = fields; return this; }

  where(condition) {
    this.#query.conditions.push(condition);
    return this;
  }

  limit(n) { this.#query.limit = n; return this; }

  build() {
    let sql = `SELECT ${this.#query.fields.join(', ')} FROM ${this.#query.table}`;
    if (this.#query.conditions.length)
      sql += ` WHERE ${this.#query.conditions.join(' AND ')}`;
    if (this.#query.limit)
      sql += ` LIMIT ${this.#query.limit}`;
    return sql;
  }
}

// ✅ Fluent API — dễ đọc như văn xuôi
const sql = new QueryBuilder()
  .from('users')
  .select('id', 'name', 'email')
  .where('age > 18')
  .where('active = true')
  .limit(10)
  .build();
// "SELECT id, name, email FROM users WHERE age > 18 AND active = true LIMIT 10"

Prototype
Clone object thay vì tạo mới từ đầu
+
Mô tả

Cho phép sao chép (clone) object hiện có mà không phụ thuộc vào class của nó. Hữu ích khi việc tạo object mới tốn kém hơn việc copy từ một prototype.

Khi nào dùng
Object khởi tạo tốn kém (query DB, parse file) — clone nhanh hơn tạo mới.
Game: spawn nhiều enemy cùng loại từ 1 template.
Undo system: lưu snapshot state bằng cách clone object.
Ví dụ code
JavaScript
class Enemy {
  constructor(type, hp, speed, skills = []) {
    this.type = type;
    this.hp = hp;
    this.speed = speed;
    this.skills = skills;
  }

  clone() {
    // Deep clone để tránh shared reference
    return new Enemy(
      this.type, this.hp, this.speed,
      [...this.skills]
    );
  }
}

// Tạo prototype một lần
const orcTemplate = new Enemy('Orc', 100, 5, ['slash', 'block']);

// Spawn 3 orc từ prototype — không cần query DB lại
const wave = Array.from({ length: 3 }, () => orcTemplate.clone());
wave[0].hp = 150; // boss orc — chỉ thay đổi clone
🔮
Abstract Factory
Tạo họ các object liên quan mà không phụ thuộc class cụ thể
+
Mô tả

Cung cấp interface để tạo một họ các object liên quan mà không cần chỉ định class cụ thể. Giống Factory Method nhưng cho nhiều loại product cùng lúc.

Khi nào dùng
UI cross-platform: tạo Button + Checkbox + Input đồng bộ cho Windows/Mac/Linux.
Hệ thống theme: Dark/Light — tất cả component phải match với nhau.
Ví dụ code
JavaScript
// Abstract products
class Button { render() {} }
class Input  { render() {} }

// Concrete products — Windows
class WinButton extends Button {
  render() { return '[Windows Button]'; }
}
class WinInput extends Input {
  render() { return '[Windows Input]'; }
}

// Abstract Factory
class UIFactory {
  createButton() {}
  createInput()  {}
}

// Concrete Factory
class WindowsFactory extends UIFactory {
  createButton() { return new WinButton(); }
  createInput()  { return new WinInput(); }
}

// Client không biết platform cụ thể
function buildForm(factory) {
  const btn = factory.createButton();
  const inp = factory.createInput();
  return [btn.render(), inp.render()];
}

buildForm(new WindowsFactory()); // ['[Windows Button]', '[Windows Input]']
Structural

Nhóm Cấu Trúc

Các pattern liên quan đến cách tổ chức class và object thành các cấu trúc lớn hơn, đồng thời vẫn giữ tính linh hoạt và hiệu quả.

🔌
Adapter
Chuyển đổi interface không tương thích
+
Mô tả

Adapter hoạt động như một lớp bọc, chuyển đổi interface của một class thành interface khác mà client mong đợi. Cho phép các class có interface không tương thích làm việc cùng nhau.

Khi nào dùng
Tích hợp thư viện bên thứ ba có API khác với hệ thống hiện tại.
Legacy code: wrap API cũ để dùng với code mới.
Chuyển đổi format dữ liệu: XML → JSON, USD → VND.
Ví dụ code
JavaScript
// Thư viện cũ: dùng XML
class OldPaymentAPI {
  payWithXML(xmlData) {
    console.log(`Paying with XML: ${xmlData}`);
  }
}

// Hệ thống mới dùng JSON
class PaymentAdapter {
  constructor(oldAPI) { this.oldAPI = oldAPI; }

  pay(jsonData) {
    // Convert JSON → XML rồi gọi API cũ
    const xml = this.jsonToXML(jsonData);
    this.oldAPI.payWithXML(xml);
  }

  jsonToXML(json) {
    return `<payment><amount>${json.amount}</amount></payment>`;
  }
}

// ✅ Client chỉ biết interface mới (JSON)
const adapter = new PaymentAdapter(new OldPaymentAPI());
adapter.pay({ amount: 500000, currency: 'VND' });

Decorator
Thêm tính năng vào object mà không thay đổi class
+
Mô tả

Cho phép thêm hành vi mới vào object bằng cách bọc nó trong một decorator object, thay vì dùng kế thừa. Có thể stack nhiều decorator lên nhau như "layers".

Khi nào dùng
Thêm tính năng vào object mà không muốn ảnh hưởng các object khác cùng class.
Middleware pipeline: logging → auth → rate-limit → handler.
I/O streams: BufferedReader bọc FileReader bọc NetworkReader.
Ví dụ code
JavaScript
class Coffee {
  cost()   { return 20000; }
  desc()   { return 'Espresso'; }
}

// Base decorator
class CoffeeDecorator {
  constructor(coffee) { this.coffee = coffee; }
  cost() { return this.coffee.cost(); }
  desc() { return this.coffee.desc(); }
}

class MilkDecorator extends CoffeeDecorator {
  cost() { return super.cost() + 5000; }
  desc() { return super.desc() + ' + Sữa'; }
}

class SyrupDecorator extends CoffeeDecorator {
  cost() { return super.cost() + 8000; }
  desc() { return super.desc() + ' + Syrup'; }
}

// ✅ Stack decorators linh hoạt
let order = new Coffee();
order = new MilkDecorator(order);
order = new SyrupDecorator(order);

console.log(order.desc()); // Espresso + Sữa + Syrup
console.log(order.cost()); // 33000
🏛️
Facade
Giao diện đơn giản cho hệ thống phức tạp
+
Mô tả

Cung cấp một interface đơn giản cho một tập hợp các interface phức tạp trong một subsystem. Facade không ẩn subsystem mà chỉ cung cấp một lớp "lối vào" dễ dùng hơn.

Khi nào dùng
SDK/Library: expose API đơn giản che giấu logic phức tạp bên trong.
Khởi động ứng dụng: 1 method start() thay vì gọi 10 service theo thứ tự.
Payment gateway: 1 charge() wrap nhiều bước: validate → tokenize → charge → log.
Ví dụ code
JavaScript
// Các subsystem phức tạp
class VideoDecoder  { decode(f)  { return `decoded(${f})`; } }
class AudioMixer    { mix(d)    { return `mixed(${d})`; } }
class Renderer      { render(d) { return `rendered(${d})`; } }

// Facade — giao diện đơn giản
class VideoPlayerFacade {
  constructor() {
    this.decoder  = new VideoDecoder();
    this.mixer    = new AudioMixer();
    this.renderer = new Renderer();
  }

  play(file) {
    const decoded  = this.decoder.decode(file);
    const mixed    = this.mixer.mix(decoded);
    return this.renderer.render(mixed);
  }
}

// ✅ Client chỉ cần biết 1 method
const player = new VideoPlayerFacade();
player.play('video.mp4'); // rendered(mixed(decoded(video.mp4)))
🛡️
Proxy
Kiểm soát truy cập vào object thật
+
Mô tả

Cung cấp một object đại diện để kiểm soát truy cập vào object thật. Proxy có thể thực hiện thêm các tác vụ (caching, logging, auth) trước/sau khi gọi object thật.

Khi nào dùng
Lazy loading: chỉ tạo object nặng khi thực sự cần (Virtual Proxy).
Caching: cache kết quả API call, tránh gọi lại (Cache Proxy).
Access control: chỉ cho phép user có quyền truy cập (Protection Proxy).
Ví dụ code
JavaScript
class RealAPI {
  async fetchUser(id) {
    // Gọi thật — chậm, tốn tiền
    return { id, name: `User ${id}` };
  }
}

// Cache Proxy
class CachedAPI {
  constructor(api) {
    this.api   = api;
    this.cache = new Map();
  }

  async fetchUser(id) {
    if (this.cache.has(id)) {
      console.log('⚡ Cache hit!');
      return this.cache.get(id);
    }
    const user = await this.api.fetchUser(id);
    this.cache.set(id, user);
    return user;
  }
}

// ✅ Client dùng Proxy như RealAPI
const api = new CachedAPI(new RealAPI());
await api.fetchUser(42); // gọi thật
await api.fetchUser(42); // ⚡ Cache hit!
🌳
Composite
Xử lý object đơn lẻ và nhóm object như nhau
+
Mô tả

Tổ chức object theo cấu trúc cây, cho phép client xử lý object đơn lẻ (Leaf) và tập hợp object (Composite) một cách thống nhất qua cùng một interface.

Khi nào dùng
File system: File và Folder cùng có size() — Folder tổng hợp size của các con.
UI Component tree: Button, Panel, Form — tất cả đều có render().
Tính lương: Employee đơn lẻ và Department (tổng lương nhóm) cùng interface.
Ví dụ code
JavaScript
class File {
  constructor(name, size) { this.name = name; this.size = size; }
  getSize() { return this.size; }
  print(indent = 0) {
    console.log(' '.repeat(indent) + `📄 ${this.name} (${this.size}KB)`);
  }
}

class Folder {
  constructor(name) { this.name = name; this.children = []; }
  add(child) { this.children.push(child); return this; }
  getSize() { return this.children.reduce((sum, c) => sum + c.getSize(), 0); }
  print(indent = 0) {
    console.log(' '.repeat(indent) + `📁 ${this.name}`);
    this.children.forEach(c => c.print(indent + 2));
  }
}

// ✅ Build file tree
const root = new Folder('root')
  .add(new File('index.html', 10))
  .add(new Folder('src')
    .add(new File('app.js', 50))
    .add(new File('style.css', 20)));

root.print();       // In cây thư mục
root.getSize();     // 80 — tổng tất cả file
Behavioral

Nhóm Hành Vi

Các pattern xử lý giao tiếp và phân chia trách nhiệm giữa các object — tập trung vào cách object tương tác và phân phối công việc.

👁️
Observer
Pub/Sub — thông báo nhiều object khi state thay đổi
+
Mô tả

Định nghĩa quan hệ one-to-many: khi một object (Subject) thay đổi trạng thái, tất cả các object phụ thuộc (Observer) tự động được thông báo. Đây là nền tảng của Event System và Reactive Programming.

Khi nào dùng
Event system: click, keypress, custom events.
State management: Redux store notify component khi state thay đổi.
Real-time: server push thông báo đến nhiều client.
Ví dụ code
JavaScript
class EventEmitter {
  #events = {};

  on(event, listener) {
    (this.#events[event] ??= []).push(listener);
    return () => this.off(event, listener); // unsubscribe fn
  }

  off(event, listener) {
    this.#events[event] = (this.#events[event] ?? [])
      .filter(l => l !== listener);
  }

  emit(event, data) {
    (this.#events[event] ?? []).forEach(l => l(data));
  }
}

// ✅ Sử dụng
const store = new EventEmitter();

const unsub = store.on('cart:update', ({ items }) => {
  console.log(`🛒 Giỏ hàng: ${items.length} sản phẩm`);
});

store.emit('cart:update', { items: ['Áo', 'Quần'] });
// 🛒 Giỏ hàng: 2 sản phẩm

unsub(); // Hủy đăng ký
♟️
Strategy
Đóng gói thuật toán, hoán đổi tự do
+
Mô tả

Định nghĩa một họ các thuật toán, đóng gói từng cái lại, và làm cho chúng hoán đổi được. Strategy cho phép thuật toán thay đổi độc lập với client sử dụng nó.

Khi nào dùng
Nhiều cách sort: BubbleSort, QuickSort, MergeSort — đổi runtime.
Payment: thay đổi phương thức thanh toán (Momo, ZaloPay, Card) mà không sửa logic đặt hàng.
Compression: ZIP, RAR, GZIP — cùng interface compress(data).
Ví dụ code
JavaScript
// Strategies
const MomoStrategy = {
  pay(amount) { console.log(`💜 Momo: ${amount.toLocaleString()}đ`); }
};

const CardStrategy = {
  pay(amount) { console.log(`💳 Card: ${amount.toLocaleString()}đ`); }
};

// Context
class Checkout {
  #strategy;

  setStrategy(strategy) { this.#strategy = strategy; }

  processPayment(amount) {
    if (!this.#strategy) throw new Error('Chọn phương thức thanh toán');
    this.#strategy.pay(amount);
  }
}

// ✅ Đổi strategy runtime
const checkout = new Checkout();

checkout.setStrategy(MomoStrategy);
checkout.processPayment(250000); // 💜 Momo: 250,000đ

checkout.setStrategy(CardStrategy);
checkout.processPayment(250000); // 💳 Card: 250,000đ
📋
Command
Đóng gói request thành object — hỗ trợ Undo/Redo
+
Mô tả

Biến một request thành một object độc lập chứa toàn bộ thông tin về request đó. Cho phép queue commands, log, undo/redo, và thực thi trễ.

Khi nào dùng
Undo/Redo: text editor, Photoshop — mỗi hành động là 1 command có thể reverse.
Task queue: gom nhiều lệnh lại thực thi theo batch.
Macro: record chuỗi lệnh, replay khi cần.
Ví dụ code
JavaScript
class TextEditor {
  text = '';
  write(s) { this.text += s; }
  delete(n) { this.text = this.text.slice(0, -n); }
}

// Commands
class WriteCommand {
  constructor(editor, text) { this.editor = editor; this.text = text; }
  execute() { this.editor.write(this.text); }
  undo()    { this.editor.delete(this.text.length); }
}

// Invoker với undo stack
class History {
  #stack = [];
  execute(cmd) { cmd.execute(); this.#stack.push(cmd); }
  undo() { this.#stack.pop()?.undo(); }
}

// ✅ Undo/Redo
const editor  = new TextEditor();
const history = new History();

history.execute(new WriteCommand(editor, 'Hello'));
history.execute(new WriteCommand(editor, ' World'));
console.log(editor.text); // "Hello World"

history.undo();
console.log(editor.text); // "Hello"
🔄
State
Thay đổi hành vi object khi state thay đổi
+
Mô tả

Cho phép object thay đổi hành vi khi trạng thái nội tại của nó thay đổi. Object sẽ trông như thể đã đổi class. Thay thế cho chuỗi if-else kiểm tra state.

Khi nào dùng
Finite State Machine: đèn giao thông, vending machine, workflow.
Media player: Playing/Paused/Stopped — mỗi state xử lý press() khác nhau.
Order status: Pending → Processing → Shipped → Delivered.
Ví dụ code
JavaScript
class TrafficLight {
  #states = {
    red:   { next: 'green',  label: '🔴 DỪNG',  duration: 3000 },
    green: { next: 'yellow', label: '🟢 ĐI',    duration: 2000 },
    yellow:{ next: 'red',    label: '🟡 CHUẨN BỊ', duration: 1000 },
  };

  #current = 'red';

  getState() { return this.#states[this.#current]; }

  next() {
    this.#current = this.#states[this.#current].next;
    console.log(this.getState().label);
  }
}

// ✅
const light = new TrafficLight();
light.next(); // 🟢 ĐI
light.next(); // 🟡 CHUẨN BỊ
light.next(); // 🔴 DỪNG
⛓️
Chain of Responsibility
Chuyền request qua chuỗi handlers
+
Mô tả

Tạo một chuỗi các handler, mỗi handler quyết định xử lý request hoặc chuyền cho handler tiếp theo. Tách sender khỏi receiver.

Khi nào dùng
Middleware stack: Express.js app.use() — mỗi middleware có thể next() hoặc dừng.
Validation pipeline: check auth → check schema → check business rules.
Support ticket: L1 → L2 → L3 support escalation.
Ví dụ code
JavaScript
class Handler {
  #next = null;
  setNext(h) { this.#next = h; return h; }
  handle(req) { return this.#next?.handle(req); }
}

class AuthHandler extends Handler {
  handle(req) {
    if (!req.token) { console.log('❌ Unauthorized'); return; }
    console.log('✅ Auth OK');
    return super.handle(req);
  }
}

class RateLimitHandler extends Handler {
  handle(req) {
    if (req.rps > 100) { console.log('❌ Rate limit'); return; }
    console.log('✅ Rate OK');
    return super.handle(req);
  }
}

class BusinessHandler extends Handler {
  handle(req) { console.log('🚀 Xử lý request thành công!'); }
}

// ✅ Build pipeline
const auth = new AuthHandler();
auth.setNext(new RateLimitHandler()).setNext(new BusinessHandler());

auth.handle({ token: 'abc', rps: 50 });
// ✅ Auth OK → ✅ Rate OK → 🚀 Xử lý request thành công!
📐
Template Method
Định khung thuật toán, để subclass điền chi tiết
+
Mô tả

Định nghĩa khung (skeleton) của một thuật toán trong base class, để subclass override một số bước cụ thể mà không thay đổi cấu trúc tổng thể.

Khi nào dùng
Data processing pipeline: đọc → xử lý → xuất — bước giữa khác nhau theo format.
Test framework: setUp() → test() → tearDown() — framework gọi theo thứ tự cố định.
Report generator: khung report giống nhau, chỉ phần data binding khác.
Ví dụ code
JavaScript
class DataExporter {
  // Template method — khung cố định
  export(data) {
    const validated  = this.validate(data);
    const formatted  = this.format(validated);
    const compressed = this.compress(formatted);
    this.save(compressed);
  }

  validate(d) { return d.filter(Boolean); } // chung
  compress(d) { return d; }                  // hook — override nếu cần

  format(d) { throw new Error('Implement format()'); }
  save(d)   { throw new Error('Implement save()'); }
}

class CSVExporter extends DataExporter {
  format(rows) { return rows.map(r => r.join(',')).join('\n'); }
  save(d)      { console.log(`💾 Saved CSV:\n${d}`); }
}

// ✅
new CSVExporter().export([['Alice', 25], ['Bob', 30]]);

Iterator
Duyệt collection mà không lộ cấu trúc nội tại
+
Mô tả

Cung cấp cách duyệt qua các phần tử của một collection mà không cần biết cấu trúc bên trong của nó (Array, Tree, Graph, LinkedList...).

Khi nào dùng
Custom data structures: LinkedList, Tree cần for...of support.
Lazy evaluation: chỉ tính giá trị khi được yêu cầu (vô hạn sequence).
Thống nhất cách duyệt nhiều loại collection khác nhau.
Ví dụ code
JavaScript
class Range {
  constructor(start, end, step = 1) {
    this.start = start;
    this.end   = end;
    this.step  = step;
  }

  // Implement Symbol.iterator → hỗ trợ for...of
  [Symbol.iterator]() {
    let current = this.start;
    const { end, step } = this;

    return {
      next() {
        if (current <= end) {
          const value = current;
          current += step;
          return { value, done: false };
        }
        return { value: undefined, done: true };
      }
    };
  }
}

// ✅ Dùng như built-in iterable
for (const n of new Range(1, 10, 2)) {
  console.log(n); // 1, 3, 5, 7, 9
}

const arr = [... new Range(1, 5)]; // [1,2,3,4,5]
